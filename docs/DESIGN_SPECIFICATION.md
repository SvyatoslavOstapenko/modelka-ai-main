## 1. Обзор проекта

### 1.1 Цель приложения

**Modelka.AI** - это веб-приложение для виртуальной примерки одежды с использованием искусственного интеллекта. Пользователи загружают фото модели (человека) и фото одежды, после чего AI генерирует реалистичное изображение одежды на модели.

### 1.2 Целевая аудитория

- **Продавцы на маркетплейсах** (Wildberries, Ozon) - создание карточек товаров
- **Интернет-магазины одежды** - визуализация товаров
- **Модельеры и дизайнеры** - презентация коллекций
- **Обычные пользователи** - виртуальная примерка перед покупкой

### 1.3 Ключевые особенности

- ✨ **Быстрая генерация** - результат за 15-30 секунд
- 🎯 **Высокое качество** - сохранение текстур, узоров, посадки одежды
- 🔐 **Безопасность** - NSFW-фильтры, защищенная загрузка
- 💳 **Кредитная система** - оплата по факту использования
- 📱 **Адаптивный дизайн** - от смартфонов до десктопов

### 1.4 Техническая архитектура

- **Frontend:** Next.js 16 (App Router) + React 19.2
- **Styling:** Tailwind CSS 4 + shadcn/ui
- **AI Provider:** Fashn.ai Virtual Try-On v1.6
- **Хранилище:** Yandex S3
- **База данных:** PostgreSQL 17

### 3.1 Карта навигации

```
┌─────────────────────────────────────────────────┐
│                  HEADER                          │
│  [Logo] [Примерка] [Галерея] [Кредиты] [Профиль]│
└─────────────────────────────────────────────────┘
                        │
        ┌───────────────┼───────────────┐
        │               │               │
    ┌───▼───┐       ┌───▼───┐      ┌───▼────┐
    │Landing│       │  /app │      │/gallery│
    │   /   │       │       │      │        │
    └───────┘       └───┬───┘      └────────┘
                        │
            ┌───────────┼──────────┐
            │                      │
       ┌────▼─────┐         ┌──────▼──────┐
       │ Форма    │         │  Результат  │
       │ загрузки │────────▶│  генерации  │
       └──────────┘         └─────────────┘
```

### 3.2 Основные маршруты

| Путь | Название | Доступ | Описание |
|------|----------|--------|----------|
| `/` | Landing Page | Публичный | Промо-страница, вход |
| `/app` | Try-On Form | Требует auth | Форма виртуальной примерки |
| `/gallery` | Gallery | Требует auth | История генераций |
| `/auth/signin` | Sign In | Публичный | Страница входа (fallback) |

### 3.3 Модальные окна

1. **Auth Modal** - Вход через email (OTP) или OAuth (Google/Yandex)
2. **Name Modal** - Сбор имени при первом входе (для OAuth пользователей)
3. **Error Dialog** - Критические ошибки (опционально)

## 7. Состояния и обработка ошибок

### 7.1 Generation Status States

| Status | Цвет badge | Progress % | Описание | UI |
|--------|-----------|------------|----------|-----|
| PENDING | outline (gray) | 10% | Подготовка задачи | Loader + "Подготовка..." |
| QUEUED | secondary (blue) | 30% | В очереди Fashn.ai | Loader + "В очереди..." |
| PROCESSING | secondary (blue) | 60% | Генерация изображения | Loader + "Генерация..." |
| COMPLETED | default (green) | 100% | Успешно завершено | Результат + кнопки |
| FAILED | destructive (red) | 0% | Ошибка генерации | Ошибка + retry |
| CANCELED | outline (gray) | 0% | Отменено пользователем | (будущее) |

### 7.2 Upload States (SmartUploader)

| State | Описание | UI |
|-------|----------|-----|
| idle | Ожидание загрузки | Drop zone с иконкой Upload |
| compressing | Сжатие изображения | Progress bar + "Сжатие..." |
| uploading | Загрузка в S3 | Progress bar + "Загрузка..." |
| success | Загружено успешно | Превью + badge "Загружено" |
| error | Ошибка загрузки | AlertCircle + кнопка retry |

### 7.3 Типы ошибок Fashn.ai

| Error Code | User Message | UI Hint |
|------------|--------------|---------|
| NSFW_DETECTED | Изображение отклонено системой безопасности. Используйте другое фото. | Попробовать с другим фото |
| No Face Detected | Лицо не найдено на фото. Загрузите фото человека анфас (хотя бы по пояс). | Загрузить фото с лицом |
| ImageLoadError | Не удалось загрузить ваши исходные изображения. Попробуйте еще раз. | Повторить загрузку |
| Too Many Requests | Высокая нагрузка на сервис. Попробуйте через минуту. | Подождать 60 секунд |
| Internal Error | Сервис временно недоступен. Попробуйте позже. | Повторить позже |

### 7.4 Toast Notifications

### 8.1 Breakpoints

Используются стандартные Tailwind breakpoints:

```css
/* Mobile */

```

### 8.2 Responsive Patterns

#### Layout Container
```tsx
<div className="container max-w-6xl mx-auto py-12 px-4">
  {/* Контент */}
</div>
```
- Container: адаптивная ширина с `max-w-*`
- Padding: `px-4` (мобильный), автоматически увеличивается

#### Grid Layouts

**2-колоночная форма (Uploader'ы):**
```tsx
<div className="grid md:grid-cols-2 gap-6">
  <SmartUploader assetType="uploaded_model" />
  <SmartUploader assetType="uploaded_garment" />
</div>
```
- Mobile: 1 колонка (вертикально)
- Tablet+: 2 колонки (горизонтально)

**3-колоночная галерея:**
```tsx
<div className="grid md:grid-cols-2 lg:grid-cols-3 gap-6">
  {/* Gallery cards */}
</div>
```
- Mobile: 1 колонка
- Tablet: 2 колонки
- Desktop: 3 колонки

#### Typography Responsive

```tsx
// Hero заголовок
<h1 className="text-4xl md:text-6xl md:text-7xl font-bold">
  Профессиональные фото
</h1>
```
- Mobile: `text-4xl` (36px)
- Tablet: `text-6xl` (60px)
- Desktop: `text-7xl` (72px)

#### Navigation Responsive

```tsx
// Header Navigation
<nav className="hidden md:flex items-center gap-1">
  {/* Nav items */}
</nav>
```
- Mobile: скрыто (`hidden`)
- Desktop: показано (`md:flex`)
- Мобильное меню: **TODO** (hamburger menu в будущем)

#### Credits Display

```tsx
<div className="hidden sm:flex items-center gap-2">
  <Coins className="w-4 h-4" />
  <span>{credits}</span>
</div>
```
- Mobile: скрыто
- Small screens+: показано

### 8.3 Mobile-First Considerations

#### Touch Targets
- Минимальный размер кнопки: `44px × 44px` (Apple HIG рекомендация)
- Button размеры: `sm` (36px), `default` (40px), `lg` (44px)

#### Drag & Drop на мобильных
- SmartUploader поддерживает как drag&drop, так и клик
- На мобильных drag&drop не работает → используется только клик

#### Модальные окна на мобильных
- Auth Modal: занимает почти весь экран на мобильных
- Name Modal: компактный, но центрирован

## 9. Анимации и микроинтеракции

### 9.1 Tailwind Animations

```css
/* Встроенные */
animate-spin       /* Loader спиннер */

transition-shadow  /* Тени (hover на карточках) */

```

### 9.2 Key Animations

#### 1. Processing Animation (GenerationResult)

```tsx
<div className="relative w-24 h-24 mx-auto">
  {/* Пульсирующий внешний круг */}
  <div className="absolute inset-0 bg-primary/20 rounded-full animate-ping" />

  {/* Статичный внутренний круг */}
  <div className="relative w-full h-full bg-primary/10 rounded-full flex items-center justify-center">
    {/* Вращающийся спиннер */}
    <Loader2 className="w-12 h-12 text-primary animate-spin" />
  </div>
</div>
```

**Эффект:** Комбинация пинга (расширение) и спиннера (вращение)

#### 2. Upload Pulse (SmartUploader idle)

```tsx
<Upload className="w-12 h-12 text-muted-foreground animate-pulse" />
```

**Эффект:** Плавная пульсация иконки

#### 3. Progress Bar Animation

```tsx
<Progress value={progress} className="h-2" />
```

**Эффект:** Плавное заполнение прогресс-бара
**Implementation:** CSS transition на width

#### 4. Hover Effects

**Button Hover:**
```css
.button:hover {
  /* Tailwind генерирует: */
  background-color: hsl(var(--primary) / 0.9);
  transition: background-color 0.2s;
}
```

**Card Hover:**
```tsx
<Card className="hover:shadow-lg transition-shadow">
```

**Link Hover:**
```tsx

```

#### 5. Toast Slide-In

**Sonner toasts** имеют встроенную анимацию:
- Появление: slide-in from bottom-right
- Исчезновение: fade-out + slide-out

### 9.3 Duration Recommendations

```css
/* Быстрые интеракции (hover, focus) */

/* Стандартные (открытие модалок, transitions) */

duration-300  /* 300ms */

/* Медленные (fade-out, closing animations) */
duration-500  /* 500ms */
```

### 9.4 Micro-interactions

#### SmartUploader Drag Over
```tsx
onDragOver={() => setIsDragOver(true)}
onDragLeave={() => setIsDragOver(false)}

// Визуальный feedback:

  'border-2 border-dashed',
```

```tsx
  }}

```

**Финальные заметки для дизайнера:**

1. **Консистентность превыше всего:** используйте единую систему компонентов (shadcn/ui) на всех экранах
2. **Mobile-first:** начинайте дизайн с мобильной версии, затем масштабируйте на desktop
3. **Фокус на состояниях:** каждый компонент должен иметь все необходимые состояния (idle, loading, success, error)
4. **Accessibility:** контрастность текста минимум 4.5:1, размер touch targets минимум 44px
5. **Реалистичность:** используйте реальные примеры фото и сгенерированных результатов для мокапов
